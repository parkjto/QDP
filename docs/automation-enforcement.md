# 자동화 강제 규칙

아래 규칙은 오류 재발 방지를 위해 항상 적용한다.

1. 오류 또는 실패가 발생하면 같은 턴에서 즉시 `docs/error-log.md`에 원인/해결/재발방지를 기록한다.
2. 오류 성격에 맞는 검증 항목을 `docs/bug-prevention-checklist.md`에 추가한다.
3. 수정 후 최소 `scan:flow` + `build`를 재실행해 통과를 확인한다.
4. 핵심 로직 변경(파서/상태/플로우)은 `quality:gate`까지 통과시켜야 완료로 본다.
5. 문서 반영 없이 코드만 수정하는 작업은 금지한다.
