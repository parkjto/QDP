# 02. 실수 및 아쉬운 점

## 1) 구조 변경 시 플로우 스크립트 누락
- 단계형 UI 도입 후 `scan:flow`가 기존 흐름을 따라가며 실패
- 원인: UI 구조 변경 시 자동화 스크립트 동기 업데이트 누락

## 2) 상태 제거 과정의 회귀
- 헤더 문구 제거 시 `message` 상태 참조가 일부 남아 빌드 에러 발생
- 원인: UI 삭제 시 상태/로직 의존성 정리 누락

## 3) 실업로드 검증 우선순위 지연
- 초기에 우회 검증을 먼저 수행해 실제 업로드 추출 실패 발견이 늦어짐
- 원인: “실제 사용자 경로” 검증 우선순위가 낮았음

## 4) 단계별 디자인 통일 지연
- 학습 타입만 먼저 개선하고 문제 수/출제 순서를 같은 톤으로 즉시 통일하지 못함
- 원인: 섹션 단위 완료 기준(동일성 체크리스트) 부재
