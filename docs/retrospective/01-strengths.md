# 01. 잘한 점

## 1) 요구사항 반영 속도
- 사용자 피드백(문구 길이, 아이콘 톤, 버튼 위치, 단계형 UX)을 빠르게 UI에 반영함
- 작은 수정 요청도 코드 반영 후 빌드 확인까지 즉시 수행함

## 2) 문제 분리 구조
- PDF 업로드/출제 설정/문제 풀이를 단계로 분리해 사용성 개선
- 문제집 단위 저장(번들)로 데이터 혼합 이슈를 방지

## 3) 자동 검증 기반 개발
- `scripts/full-flow-scan.mjs`로 버튼 플로우 자동 점검 체계 구축
- 테스트/빌드 검증 루틴화로 변경 안정성 확인 속도 개선

## 4) 보안/안정성 개선
- PDF 검증(타입/용량/헤더) 추가
- 내부 예외 노출을 줄이기 위해 사용자 친화 오류 메시지 체계 적용
