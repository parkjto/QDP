# 03. 재발 방지 규칙

## 규칙 1) 변경 시 3종 동기화
아래 3개가 모두 반영되어야 완료로 간주:
1. UI 코드
2. 자동화 스크립트 (`scan:flow`)
3. 문서 (`button-flow`, `error-log`, `checklist`)

## 규칙 2) 검증 우선순위
1. 실사용 경로 검증(실제 PDF 업로드)
2. 자동 스캔 검증(`npm run scan:flow`)
3. 단위/회귀 테스트(`npm run test`)
4. 빌드 검증(`npm run build`)

## 규칙 3) 디자인 완료 기준
- 동일 유형 카드/섹션은 높이, 간격, 타이포, 상태색이 모두 일치해야 함
- 한 섹션만 개선된 상태는 임시 상태로 분류하고 후속 통일을 즉시 수행

## 규칙 4) 피드백 반영 규칙
- 문구 길이/톤/배치 요청은 즉시 반영
- 반영 후 반드시 실화면 검증(자동 스캔 + 수동 시각 점검) 수행
